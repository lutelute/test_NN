Grokking現象とは

Grokking（グロッキング）は、SF作家ロバート・A・ハインラインの小説『異星の客』(1961) に登場する火星語で、「深く直感的に理解する」という意味です。

機械学習では、過学習状態が長期間続いた後、突然汎化が起こる現象を指します。

通常の学習 vs Grokking

フェーズ 通常の学習 Grokking
初期 Train/Test共に上昇 Trainのみ上昇
中期 両方が収束 Train 100%, Test ~0%
後期 （終了） 突然 Test が急上昇!

なぜ不思議なのか

  • 過学習 = 悪という常識に反する
  • 早期停止したら汎化しない
  • 訓練100%のずっと後にテスト精度が上がる

モジュラー加算タスク

モジュラー演算は「割った余り」の演算です。

通常の加算:    5 + 3 = 8
モジュラー加算: (5 + 3) mod 7 = 1  （8÷7の余り）

時計の例:
現在10時、5時間後 → (10 + 5) mod 12 = 3時

本プロジェクトの設定

  • 入力: (a, b) where a, b ∈ {0, ..., 96}
  • 出力: (a + b) mod 97
  • 訓練データ: 全ペアの30%

フーリエ表現理論

Grokking後、モデルは数値をフーリエ基底で埋め込みます：

e(n) ≈ [cos(2πkn/p), sin(2πkn/p)]

この表現では、数値が円周上に等間隔で配置されます。

可視化の見方

1. 円環構造（左パネル）

状態 見た目 Circle値

記憶段階 不規則なクラスタ 0.3 - 0.7
Grokking後 きれいな円環 > 0.9

2. ニューロン相関行列（7×7グリッド）

  • 円形パターン: cos/sinペア（フーリエ基底）
  • 直線パターン: 強い相関
  • ランダム散布: 独立した次元

3. 学習曲線

  • 青線: Train Accuracy
  • 赤線: Test Accuracy
  • 黄色縦線: 現在のエポック

使い方

インストール

git clone https://github.com/lutelute/test_NN.git
cd test_NN
pip install -r requirements.txt

学習の実行

# 10エポック刻みで学習
python train_grokking_demo.py --linear --save_interval 10

# 2エポック刻みで詳細学習
python train_grokking_demo.py --linear --save_interval 2 --checkpoint_dir checkpoints_demo_2ep

可視化ダッシュボード

streamlit run interactive_dashboard.py

ブラウザで http://localhost:8501 を開く
